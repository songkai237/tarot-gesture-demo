# 塔罗牌专属详情内容设计

## 目标

为完整 78 张塔罗牌提供独立、可阅读的详情内容，避免小阿卡那沿用花色与数字组合的通用文案。

## 数据结构

每张牌保留现有的关键词、正位释义、逆位释义和今日指引，并新增以下字段：

- 象征意象：牌面核心符号或能量的简洁解释。
- 爱情：正位与逆位各一条独立解读。
- 事业：正位与逆位各一条独立解读。
- 行动建议：正位与逆位各一条可执行的当下建议。

所有文案采用引导式语气，避免承诺结果或把占卜解释写成绝对事实。

## 界面

详情面板保持现有位置和滚动行为，内容按以下顺序展示：

1. 牌名、正逆位和关键词。
2. 牌义：象征意象、正位释义、逆位释义。
3. 场景：爱情与事业；依据本次抽牌的正逆位显示对应文案。
4. 今日指引：依据本次抽牌的正逆位显示对应行动建议。

移动端延续面板内部滚动，不改变抽出的卡牌、摄像头窗口与重新洗牌按钮的位置。

## 实现边界

- 扩展 `TarotCard` 类型和 `tarotData.ts` 中的所有卡牌数据。
- 大阿卡那与小阿卡那均写入独立内容；不在运行时通过花色或数字拼接详情文案。
- 更新详情面板 DOM、渲染函数和样式，使新增信息有明确层级。
- 不修改抽牌、洗牌、手势识别或 Three.js 牌阵行为。

## 验收

- 任意抽到的 78 张牌都显示专属的象征、爱情、事业与行动建议。
- 正位和逆位会显示各自对应的爱情、事业及行动建议。
- 桌面和移动端详情内容可完整阅读且不遮挡关键操作。
- TypeScript 构建通过。
