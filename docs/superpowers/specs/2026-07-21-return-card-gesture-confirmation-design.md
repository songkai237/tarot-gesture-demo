# 放回卡牌手势确认设计

## 目标

避免已翻牌状态下，短暂的握拳识别或手部抖动立即触发卡牌放回。

## 状态流程

1. 已翻牌时，张开手掌连续保持至少 300ms，进入“待放回确认”状态。
2. 进入待确认后，用户必须先离开张掌姿势，再做出握拳；这保证动作是明确的“张手到握拳”切换。
3. 握拳连续保持 600ms，且横向手掌速度低于 `0.28`，才开始放回动画。
4. 确认期间任何松手、切换到其他手势或明显移动都会取消本次确认，回到已翻牌状态。
5. 待确认状态最长持续 3 秒；超时自动取消，避免之后的误触。

## 反馈

- 稳定张手后显示“握拳并停住以放回”。
- 握拳确认期间显示“确认放回中”。
- 取消或超时后显示“重洗：张掌后握拳”。

## 实现边界

- 手势识别器保持输出当前帧的手势、置信度和速度。
- 应用层维护确认计时与阶段状态，避免改变洗牌和抽牌的既有规则。
- 按钮“重新洗牌”仍可立即触发放回，不受手势确认延迟影响。

## 验收

- 短暂误判为握拳时，卡牌不会放回。
- 用户张手后稳定握拳约 0.6 秒，可以稳定触发放回。
- 握拳过程中移动或松开手，放回不会触发。
- 3 秒内未完成动作，系统自行取消待确认状态。
